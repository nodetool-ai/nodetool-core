## 2025-02-27 - SQL Injection Vulnerabilities in PostgresAdapter
**Vulnerability:** Dynamic DDL and DML statements (like `CREATE TABLE`, `ALTER TABLE`, `INSERT`, `SELECT`, `DELETE`) in `src/nodetool/models/postgres_adapter.py` were built using Python's `f-string` concatenation, rendering table names, column names, and indices vulnerable to SQL injection.
**Learning:** These vulnerabilities existed because raw string concatenation was used instead of the `psycopg.sql` query composition functionality that automatically manages the safe quoting of strings as identifiers or values.
**Prevention:** In `src/nodetool/models/postgres_adapter.py` (and similar files), always use `psycopg.sql.SQL`, `psycopg.sql.Identifier`, and `psycopg.sql.Placeholder` for securely generating dynamic database queries instead of raw Python strings.
