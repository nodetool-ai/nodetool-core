## 2026-03-14 - [Symlink Traversal in Path Boundary Checks]
**Vulnerability:** Path boundary checks in `resolve_workspace_path`, `ensure_within_root`, and related API functions used `os.path.abspath` instead of `os.path.realpath`. An attacker could create a symbolic link within the allowed workspace that points outside (e.g., to `/etc`), effectively bypassing the boundary restriction when the path was checked.
**Learning:** `os.path.abspath` only normalizes the path string (collapsing `..` and `.`) but does not resolve symbolic links at the OS level. If boundary checks are implemented with just `abspath` and `commonpath`, symlink traversal vulnerabilities are a near certainty.
**Prevention:** Always use `os.path.realpath` to fully resolve symlinks before validating that a given path is contained within a permitted root directory.
