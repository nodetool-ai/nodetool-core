## 2025-04-20 - [Performance] get_content string concatenation vs get_files os.listdir
**Learning:** For recursively finding files in Python, `os.walk` is significantly faster than writing a custom recursive function using `os.listdir()` and checking `os.path.isdir()` on each item, because `os.walk()` uses `os.scandir()` internally and avoids a separate `stat` system call for every file in the tree. Also, contrary to typical Python string concatenation performance knowledge where `+=` has $O(n^2)$ overhead and `.append()` with `"".join()` is faster, using `.append()` inside the file loop for string concatenation in `get_content` (`src/nodetool/io/get_files.py`) was actually slightly slower during benchmarks compared to just iteratively adding to a string when the files themselves are very large. This might be due to `append()` having to allocate the large read-in string chunks in the list memory anyway.
**Action:** Default to `os.walk()` or `os.scandir()` instead of recursive `os.listdir()` and `os.path.isdir()` when traversing directory trees to minimize repeated filesystem metadata lookups. Be wary of blindly refactoring working string concatenations (`+=`) to `.join()` lists without benchmarking when doing chunked file processing, as it isn't a guaranteed win in this specific environment.
