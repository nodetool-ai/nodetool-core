## 2025-02-12 - Graph Property Caching Optimization
**Learning:** Frequent property access on Pydantic models with list comprehensions across relations (e.g. O(N) over `nodes` or O(E) over `edges`) can compound into significant overhead when invoked frequently (such as during actor event execution). Furthermore, when adding caches tied to collection lengths (like `_cached_edges_len`), updating them individually per cache can cause out-of-sync cache bugs if the underlying collection is modified.
**Action:** Always prefer lazy `_cache` variables for computational properties (`inputs`, `outputs`, `get_control_edges`). When multiple caches depend on the same external sequence (like `edges`), update them atomically in a single rebuild step to guarantee consistent state across independent queries.
